# Project Intelligence: Rational Physics

This document provides a distilled overview of the project's architecture, tooling, and documentation standards to orient AI agents and developers.

## 1. Project Context
The project models Stochastic Quantum Theory using NJ Wildberger's Box Arithmetic, Chromogeometry and Rational Trigonometry. It uses **Idris 2** for formal verification and **Quarto** for generating a DPF book with the "Orange Book" template.

## 2. Core Technology Stack
- **Language**: Idris 2 (v0.7.0+).
- **Package Manager**: `pack` (Idris 2 package manager).
- **Documentation**: Quarto with the `orange-book-typst` template.
- **Environment**: All builds and tests MUST be run within the Fedora toolbox container (`toolbox run -c fedora-toolbox-44`) to ensure Chez Scheme and Idris 2 dependencies are met.

## 3. The "Verified Documentation" Pipeline (Orchestrated)
The project follows a "Verified Documentation" pattern, where the book is only updated if all chapters pass their respective test suites. This is managed centrally by the **`idris2-book`** library:
1.  **Source of Truth**: Literate Markdown (`.md`) test files located in each chapter's `[project-name]-tests/` directory.
2.  **Orchestration**: The **`idris2-book`** project acts as the master build system. It maintains the chapter order in `src/Book.idr`.
3.  **The Master Command**: Running `pack test idris2-book` triggers the entire pipeline:
    - **Verification**: It iterates through every chapter and runs `pack test [chapter]` in its own folder.
    - **Synchronisation**: If tests pass, it syncs the literate files to `Book/Src/[project-name]/` as `.qmd` files.
    - **Configuration**: It auto-generates the `Book/_quarto.yml` file to ensure the book's structure matches the code.
    - **Rendering**: Finally, it triggers `quarto render` in the `Book/` directory.

**CRITICAL RULE**: Never edit files in `Book/Src/` or `Book/_quarto.yml` directly. They are machine-generated by `idris2-book`.

## 4. Directory Structure and Naming Conventions
- **Libraries**: Prefix with `idris2-` (e.g., `idris2-Multiset-Basic`, `idris2-Singletons-Pixels-and-Maxels`).
- **Tests**: Library internal test folder should match `[library-name]-tests`.
- **Physics Suite**: All physics module tests in `idris2-rcit/tests/physics/` MUST reside in a `standard/` subdirectory to maintain compatibility with the unified build orchestrator.
- **Book**: Central Quarto project in `Projects/Book`.
    - `Book/Src/`: Destination for synced docs, organised by project name.
    - `Book/[project-name].qmd`: Chapter file that includes the synced content via `{{< include Src/[project-name]/[module].qmd >}}`.

## 5. Standardised Tooling
- **`idris2-book`**: The central orchestrator library.
    - `src/Book.idr`: Contains the master chapter list, the `pack test` automation, and the YAML generation logic.
    - `pack test idris2-book`: The single command to verify the entire ecosystem and build the book.
- **`idris2-book-tests/`**: Contains a "Build Test" (`Main.md`) that executes the orchestrator's logic within the Idris test runner.

## 6. Dynamic Indexing
The `idris2-index` library generates the `Book/Src/idris2-index/index.qmd` preface. Like other chapters, it is verified by its own test suite before being included in the book.
