Usage: pack [options] <cmd> [<args>]

Options:

  -p <db>        --package-set=<db>          Set the curated package set to use.
                 --cg=<codgen>               Sets the backend to use when building Idris executables
                                             or running the REPL. The default is to use the `chez`
                                             code generator.
                 --scheme=<exec>             Sets the scheme executable for installing the Idris2 compiler.
                                             As a default, this is set to `scheme`.
  -P <packages>  --packages=<packages>       Load the given (comma-separated) list of packages into the REPL
                                             session.
  -s             --short-desc                Print the short description stored in an `.ipkg` file for
                                             each query result.
  -l             --long-desc                 Print a detailed description of a package known to pack
                 --tree                      Print a dependency tree of a package known to pack
                 --reverse-tree              Print a tree of packages depending on a package know to pack.
                                             Use this to find all packages transitively depending on a specific
                                             library
  -d             --dependencies              Print the dependencies of each query result.
  -o <file>      --output=<file>             Name of the output file when compiling or running single Idris source files
                                             This defaults to `_tmppack` if not specified explicitly"
                 --ipkg                      Print the full `.ipkg` file of each query result.
                 --prompt                    Prompt before installing a potentially unsafe package
                                             with custom build hooks.
                 --no-prompt                 Don't prompt before installing a potentially unsafe package
                                             with custom build hooks.
                 --gc-prompt                 Prompt before deleting directories when running command `gc`.
                 --gc-purge                  Remove *all* outdated libraries during garbage collection.
                 --no-gc-prompt              Don't prompt before deleting directories when running command `gc`.
                 --no-gc-purge               Only remove libraries built with an outdated compiler but not
                                             outdated libraries built with the current compiler during garbage
                                             collection.
                 --bootstrap                 Use the bootstrap compiler for building Idris2. This takes longer
                                             than without bootstrapping, but it will even work if no Idris2
                                             compiler or an outdated one is on the `$PATH`.
                 --no-bootstrap              Use an existing version of Idris2 when building the compiler.
                                             This will fail if `idris2` is not on the computer's `$PATH` or
                                             is too old to build the current version of the compiler.
                 --bootstrap-stage3          When bootstrapping, rebuilds the compiler once more using the newly
                                             built compiler to produce a more optimised final version.
                 --no-bootstrap-stage3       Don't perform an additional compiler rebuild during bootstrapping.
                 --warn-depends              Issue a warning in precense of a local `depends` directory, which might
                                             interfere with the libraries managed by pack.
                 --no-warn-depends           Don't issue a warning in precense of a local `depends` directory.
                 --skip-tests                Don't run library tests during collection checking.
                                             This currently only affects the pack-admin utility.
                 --with-src                  Include the source code of a library when installing
                                             it. This allows some editor plugins to jump to the
                                             definitions of functions and data types in other
                                             modules.
                 --with-docs                 Include the API documentation when installing libraries.
                 --use-katla                 Use katla to add links to the semantically highlighted API sources.
                 --with-ipkg=<.ipkg>         Use settings and packages from the given `.ipkg` file when
                                             starting a REPL session.
                 --no-ipkg                   Don't look for an `.ipkg` file in scope when starting a REPL session.
                 --rlwrap[=<rlwrap args>]    Run a REPL session in `rlwrap`.
                 --extra-args=<idris2 args>  Any extra arguments to pass to Idris2.
  -v             --verbose                   Print debugging information
  -q             --quiet                     Quiet mode
                 --log-level=<log level>     Specify the logging level to use. Accepted values are:
                                             "debug", "build", "info", "cache", "warning", "silence".
                 --git-init                  Initialize git for a pack project.


Run `pack help <cmd>` to get detailed information about a command.
